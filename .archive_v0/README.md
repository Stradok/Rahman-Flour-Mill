# Archive (v0)

Historical material kept for reference. Nothing in this directory is used by
the running application — the live documentation is the root `README.md` and
`docs/`.

| Folder | Contents |
|---|---|
| `notes/` | Development-era notes: the running client spec (`Second-ammendment.md`), idea drafts, deployment notes, session logs (`fable.md`, `windows.md`, `stuff_to_do.md`) |
| `reports/` | Point-in-time session reports (code review, delivery/feature summaries) — superseded by git history and the QA suite |
| `docs_v0/` | First-generation docs (installation, auth/update strategy, owner settings, community files) — superseded by `docs/BUILD_WINDOWS_INSTALLER.md` and `docs/Version_Controlling.md` |
| `screenshots/` | Early UI screenshots |
| `ci/` | A GitHub Actions release workflow that was never functional (wrong Node version, missing build inputs). Releases are built locally per `docs/BUILD_WINDOWS_INSTALLER.md` |
| `prisma_historical/` | Prisma schema + migrations from before the move to Drizzle + encrypted SQLite |
| `cloud.md` | Early cloud/Supabase exploration notes (the project became local-only) |
